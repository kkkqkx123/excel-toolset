## 技术术语表

| 术语 | 说明 |
|------|------|
| Agent | 智能助手主体，负责理解用户意图并执行操作 |
| Tool | 工具，Agent执行Excel操作的基本单元 |
| Skill | 技能，Agent能力的扩展和增强 |
| Prompt | 提示词，系统指令和用户指令的集合 |
| Context | 上下文，自动收集的用户操作环境信息 |
| Blueprint | 蓝图，工作簿的结构化元数据 |
| Convention | 约定，格式化和样式的一致性规范 |
| Execution Policy | 执行策略，工具执行的权限分类 |
| MCP | Model Context Protocol，模型上下文协议 |
| Pyodide | Python浏览器内WebAssembly运行时 |

## 设计模式

### 1. 工厂模式
- 工具创建：`create*Tool()` 函数
- 技能编录：`buildCatalog()` 函数

### 2. 策略模式
- 执行策略：READ_NONE, MUTATE_CONTENT, MUTATE_STRUCTURE
- 输出截断策略：head, tail

### 3. 观察者模式
- 变更追踪：监听工作簿变更
- 事件系统：技能激活和工具调用事件

### 4. 代理模式
- 工作簿协调器：代理工具执行
- 连接管理：代理外部服务访问

### 5. 装饰器模式
- 输出截断：包装工具执行
- 错误处理：包装工具执行
- 性能监控：包装工具执行

## 扩展指南

### 添加新工具

1. 在 `tools/` 目录创建工具文件
2. 实现工具定义和执行函数
3. 在 `tools/registry.ts` 中注册工具
4. 更新工具文档和提示词

### 添加新技能

1. 在 `skills/` 目录创建技能目录
2. 创建 `SKILL.md` 文件
3. 编写技能描述和内容
4. 技能会自动被发现和加载

### 添加新约定

1. 使用 `conventions` 工具的 `set` 操作
2. 定义预设格式字符串
3. 设置视觉默认值和颜色约定
4. 约定会自动注入系统提示词

### 添加新集成

1. 在 `connections/` 定义连接类型
2. 实现连接管理和认证
3. 创建对应的工具
4. 配置MCP服务器（如适用）

## 常见问题

### Q: 如何调试工具执行问题？
A: 查看工具输出的 `details` 字段，检查错误信息和执行上下文。

### Q: 如何优化工具性能？
A: 使用批量操作、实现缓存、减少不必要的同步调用。

### Q: 如何扩展系统功能？
A: 可以通过添加新工具、新技能、新集成来扩展系统功能。

### Q: 如何处理大型工作簿？
A: 使用分页读取、限制输出大小、优化查询范围。

### Q: 如何确保安全性？
A: 实现参数验证、权限检查、输入消毒、输出过滤。
