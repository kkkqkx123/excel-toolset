# Excel AI 助手系统架构分析

## 项目概述

这是一个基于TypeScript开发的Excel AI助手系统，通过Office.js API与Excel进行交互，提供智能化的Excel操作能力。系统采用模块化设计，包含工具层、技能层、上下文层和连接层等多个子模块。

## 核心架构

### 1. 工具系统 (tools/)

工具系统是整个系统的核心，提供了Excel操作的各种功能接口。

#### 1.1 工具注册机制

- **工具注册中心**：`tools/registry.ts` 负责所有核心工具的注册和管理
- **工具入口**：`tools/index.ts` 统一创建所有工具（核心工具 + 实验性工具）
- **工具类型定义**：使用TypeScript严格类型系统和TypeBox进行参数验证

#### 1.2 核心工具分类

**工作簿管理工具**
- `get_workbook_overview`: 获取工作簿结构概览
- `search_workbook`: 在工作簿中搜索内容
- `workbook_history`: 工作簿历史记录和备份恢复

**数据读写工具**
- `read_range`: 读取单元格数据，支持三种模式（compact/csv/detailed）
- `write_cells`: 写入单元格数据，支持值和公式
- `fill_formula`: 使用Excel自动填充功能填充公式

**格式化工具**
- `format_cells`: 单元格格式化（字体、颜色、边框、对齐等）
- `conditional_format`: 条件格式设置
- `format_cells_borders`: 专门的边框格式化

**结构修改工具**
- `modify_structure`: 修改工作表结构（增删行列、重命名等）

**分析工具**
- `trace_dependencies`: 追踪单元格依赖关系（前驱/后继）
- `explain_formula`: 解释公式逻辑
- `explain_formula_logic`: 深度公式逻辑分析

**辅助工具**
- `view_settings`: 视图设置
- `comments`: 批注管理
- `instructions`: 用户指令管理
- `conventions`: 格式约定设置
- `names`: 命名范围管理

#### 1.3 实验性工具

**Python执行工具**
- `python_run`: Python代码执行（支持Pyodide和本地桥接）
- `python_transform_range`: 使用Python转换单元格数据
- `libreoffice_convert`: LibreOffice文件转换

**终端工具**
- `tmux`: 终端复用器集成

**文件管理工具**
- `files`: 文件工作空间管理

**外部集成工具**
- `external_fetch`: 外部HTTP请求
- `fetch_page`: 网页内容获取
- `mcp`: Model Context Protocol网关

**Office.js直调工具**
- `execute_office_js`: 直接执行Office.js代码

**扩展管理工具**
- `extensions_manager`: 扩展生命周期管理

#### 1.4 工具特性

**安全机制**
- 输出截断：所有工具输出限制50KB和2000行
- 执行策略：分类执行权限（read/none, mutate/content, mutate/structure）
- 覆盖保护：写入数据时防止意外覆盖
- 公式验证：公式语法和引用验证

**变更追踪**
- 单元格差异：记录写入前后的变化
- 审计日志：记录所有工作簿变更操作
- 快照备份：自动创建操作前快以便恢复

**错误处理**
- 结构化错误信息：提供详细的错误上下文
- 恢复机制：操作失败时的回滚能力
- 诊断信息：丰富的调试和故障排除信息

### 2. 技能系统 (skills/)

技能系统提供Agent能力的动态扩展和发现机制。

#### 2.1 技能目录

- **技能编录**：`skills/catalog.ts` 管理所有可用技能
- **技能定义**：`skills/types.ts` 定义技能数据结构
- **技能解析**：`skills/frontmatter.ts` 解析技能文档

#### 2.2 技能发现

**内建技能**
- 从`skills/*/SKILL.md`自动发现
- 前置元数据解析（name, description, compatibility）
- 启动时自动加载

**外部技能**
- 运行时动态安装
- 通过技能工具管理
- 支持更新和卸载

#### 2.3 技能管理

**激活存储**
- `skills/activation-store.ts` 管理技能激活状态
- 会话级别和全局级别激活

**外部存储**
- `skills/external-store.ts` 外部技能存储
- 支持远程技能源

**缓存机制**
- `skills/read-cache.ts` 技能内容缓存
- 提高技能读取性能

**事件系统**
- `skills/events.ts` 技能相关事件处理

### 3. 上下文系统 (context/)

上下文系统负责自动收集和管理用户的操作上下文。

#### 3.1 选择上下文

**自动上下文读取**
- `context/selection.ts` 自动读取用户选择区域
- 提供±5行的上下文窗口
- 包含公式和错误信息

#### 3.2 工作簿蓝图

**结构元数据**
- `context/blueprint.ts` 工作簿结构蓝图
- 包含表、列、范围等结构信息

#### 3.3 变更追踪

**变更监控**
- `context/change-tracker.ts` 工作簿变更追踪
- 实时监控单元格、格式、结构变化

#### 3.4 工具披露

**工具可见性**
- `context/tool-disclosure.ts` 工具信息披露
- 控制Agent对工具的访问权限

### 4. 连接系统 (connections/)

连接系统管理外部服务连接和认证。

#### 4.1 连接管理

**连接管理器**
- `connections/manager.ts` 连接生命周期管理
- 连接状态监控和验证

**连接存储**
- `connections/store.ts` 连接凭证和配置存储
- 安全的密钥管理

#### 4.2 连接类型

**认证类型**
- API密钥认证
- Bearer Token认证
- OAuth认证
- 自定义认证

**HTTP认证**
- 头部注入
- 模板化认证信息
- 目标主机限制

### 5. 提示词系统

提示词系统是Agent行为的控制中心，包含系统提示词和用户指令。

#### 5.1 系统提示词生成

**工具披露**
- 根据可用工具动态生成工具描述
- 包含工具参数和返回格式说明

**技能集成**
- 将可用技能信息注入系统提示词
- 技能描述和使用指导

**约定注入**
- 格式约定信息注入
- 默认行为规范

#### 5.2 用户指令系统

**指令工具**
- `instructions.ts` 用户指令管理工具
- 支持全局和工作簿级别指令

**指令存储**
- 持久化存储用户自定义指令
- AGENTS.md风格的持久化指导

**指令范围**
- 用户级别：全局指令
- 工作簿级别：特定工作簿指令

#### 5.3 约定系统

**格式约定**
- `conventions.ts` 格式约定管理工具
- 预设格式字符串和样式

**约定解析**
- 数字格式约定
- 颜色约定
- 标题样式约定

### 6. 执行策略系统

执行策略系统控制工具的安全执行和资源管理。

#### 6.1 策略分类

**读操作 (read/none)**
- 只读操作，不修改工作簿
- 如：read_range, get_workbook_overview

**内容修改 (mutate/content)**
- 修改单元格内容和公式
- 如：write_cells, fill_formula

**结构修改 (mutate/structure)**
- 修改工作簿结构
- 如：modify_structure, format_cells

#### 6.2 工作簿协调器

**协调机制**
- `with-workbook-coordinator.ts` 工作簿级别的操作协调
- 防止并发冲突

**蓝图失效处理**
- 结构修改后自动失效相关缓存
- 重新加载工作簿结构

**预检机制**
- `connection-requirements.ts` 连接需求预检
- 确保执行前的资源可用性

#### 6.3 实验性工具门控

**工具门控**
- `experimental-tool-gates.ts` 实验性工具门控
- 基于配置的工具启用/禁用

**健康检查**
- 桥接服务健康检查
- 连接可用性验证

**错误处理**
- 结构化错误返回
- 用户友好的错误提示

### 7. 扩展系统

扩展系统支持第三方扩展的加载和管理。

#### 7.1 扩展管理

**扩展管理器**
- `extensions-manager.ts` 扩展生命周期管理
- 安装、启用、禁用、卸载

**扩展运行时**
- 支持内联代码和远程URL扩展
- 沙箱环境隔离

#### 7.2 扩展API

**能力桥接**
- LLM完成能力
- HTTP请求能力
- 存储访问能力
- 剪贴板操作能力

**权限控制**
- 细粒度权限管理
- 能力门控机制

**动态工具**
- 运行时工具注册和注销
- 动态工具集更新

### 8. 辅助系统

#### 8.1 Excel辅助工具

**Excel助手**
- `excel/helpers.ts` Office.js API封装
- 单元格地址解析和转换
- 范围操作工具函数

#### 8.2 工具系统

**工具详情**
- `tool-details.ts` 工具执行详情类型定义
- UI渲染所需的结构化数据

#### 8.3 变更系统

**变更管理**
- `mutation/` 目录下的变更相关模块
- 变更完成和结果记录

#### 8.4 输出管理

**输出截断**
- `output-truncation.ts` 工具输出截断
- `output-truncation-storage.ts` 截断内容存储

## 技术栈

- **语言**: TypeScript
- **Excel接口**: Office.js
- **类型验证**: TypeBox
- **运行时**: 浏览器环境
- **Python执行**: Pyodide
- **协议**: MCP (Model Context Protocol)

## 设计原则

1. **模块化设计**: 各模块职责清晰，松耦合
2. **类型安全**: 完整的TypeScript类型定义
3. **错误处理**: 结构化错误信息和恢复机制
4. **安全优先**: 多层安全防护和权限控制
5. **可扩展性**: 支持动态扩展和插件化
6. **用户体验**: 友好的交互和清晰的反馈
