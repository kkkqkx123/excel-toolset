我用**最落地、最贴合AI Agent使用**的方式，分三部分讲清楚：
1. **Excel高频核心操作**（Agent真正会用到的，剔除冷门功能）
2. **每个操作的底层原理**（吃透本质，不盲目封装）
3. **标准化封装为Agent工具的方法**（直接可落地的规则+示例）

先铺垫**核心底层原理**（所有Excel操作的根）：
我们用的 `.xlsx` 不是二进制文件，而是**ZIP压缩包**，解压后是一堆**XML文件**（存储表格结构、数据、样式、公式）。
所有Excel操作，本质都是：**解压 → 读写/修改XML节点 → 重新压缩**。
- 无头库（openpyxl/pandas）：直接操作XML，无需安装Excel
- 原生接口（xlwings）：调用Excel软件的COM引擎，间接修改XML

# 一、Excel高频核心操作 + 原理 + 封装方案
按Agent使用频率排序，每个操作包含：**功能**→**原理**→**工具封装**

## 第一类：文件/工作簿操作（基础必选）
Agent操作Excel的入口，无风险、高频使用
1. **创建新Excel文件**
   - 原理：生成空的XML结构（workbook.xml）→ 打包为 `.xlsx`
   - 封装工具：`excel_create`
     参数：文件路径、文件名
     返回：文件路径、创建状态
2. **打开/读取文件信息**
   - 原理：解压ZIP，解析根XML，获取工作表列表、文件大小、修改时间
   - 封装工具：`excel_info`
     参数：文件路径
     返回：工作表名、行数、列数、文件哈希（防篡改）
3. **保存/备份文件**
   - 原理：将修改后的XML重新压缩；备份=复制原文件
   - 封装工具：`excel_save` / `excel_backup`
     参数：原路径、备份路径
     返回：备份文件路径、操作日志
4. **关闭文件**
   - 原理：释放文件句柄，结束XML解析进程
   - 封装工具：`excel_close`

## 第二类：工作表操作（Sheet）
1. **列出所有工作表**
   - 原理：解析根XML的`<sheets>`节点，获取所有sheet名称
   - 封装工具：`excel_list_sheets`
2. **新增/删除/重命名工作表**
   - 原理：增删改XML中的sheet节点，生成/删除对应的worksheet.xml文件
   - 封装工具：`excel_add_sheet` / `excel_delete_sheet` / `excel_rename_sheet`
     参数：文件路径、工作表名、新名称
     返回：操作结果、当前sheet列表

## 第三类：单元格/区域操作（核心数据读写）
Agent最常用的核心功能，**语义化封装，禁止LLM写底层代码**
1. **读取单个/区域单元格**
   - 原理：定位到对应工作表的XML → 解析行/列节点 → 读取数据（文本存在sharedStrings.xml，数字直接存储）
   - 封装工具：`excel_read_cell` / `excel_read_range`
     参数：文件路径、工作表、单元格坐标（A1）/ 区域（A1:C5）
     返回：JSON格式数据（坐标+值+类型）
2. **写入单个/区域单元格**
   - 原理：修改XML对应节点的数值 → 标记文件为已修改
   - 封装工具：`excel_write_cell` / `excel_write_range`
     参数：文件路径、工作表、坐标、写入值
     返回：写入成功行数、修改前值（可回滚）
3. **清空单元格**
   - 原理：删除XML节点数据，保留格式
   - 封装工具：`excel_clear_range`

## 第四类：数据处理操作（Agent核心价值）
自动化分析、清洗数据，比手动操作效率高
1. **追加行/插入行/删除行**
   - 原理：在XML的`<row>`节点前后插入/删除节点，刷新行号
   - 封装工具：`excel_append_row` / `excel_insert_row` / `excel_delete_row`
     参数：数据列表、行号
     返回：影响行数
2. **筛选数据**
   - 原理：遍历XML行数据，按条件匹配 → 返回符合条件的行
   - 封装工具：`excel_filter`
     参数：筛选列、条件（>100/包含"文本"）
     返回：筛选结果列表
3. **排序数据**
   - 原理：加载数据到内存排序 → 重写XML行节点
   - 封装工具：`excel_sort`
     参数：排序列、升序/降序
4. **去重数据**
   - 原理：内存中对指定列去重 → 重写XML
   - 封装工具：`excel_deduplicate`

## 第五类：公式与计算（业务必备）
1. **设置单元格公式**
   - 原理：在XML的`<f>`标签写入公式（如`=SUM(A1:A5)`），`<v>`标签存计算值
   - 封装工具：`excel_set_formula`
     参数：坐标、公式字符串
     ✅ 关键：无头库不自动计算，xlwings可调用Excel引擎实时计算
2. **刷新计算结果**
   - 原理：触发公式重算，更新XML中的值节点
   - 封装工具：`excel_refresh_calc`

## 第六类：格式样式操作（可选）
适合生成报表，非必需
1. **设置字体/颜色/对齐/边框**
   - 原理：样式存储在styles.xml，单元格通过ID引用样式；修改样式引用关系
   - 封装工具：`excel_set_format`
     参数：区域、字体大小、颜色、对齐方式
2. **合并单元格**
   - 原理：XML中标记合并区域，只保留第一个单元格数据
   - 封装工具：`excel_merge_cells`

## 第七类：高级操作（进阶）
1. **SQL查询Excel数据**
   - 原理：用DuckDB加载Excel到内存 → 执行SQL → 返回结果
   - 封装工具：`excel_sql_query`
     参数：SQL语句
     返回：查询结果
2. **生成图表/数据透视表**
   - 原理：生成图表对应的XML节点，绑定数据区域
   - 封装工具：`excel_create_chart`

# 二、AI Agent工具封装通用规范（核心！）
所有操作必须按这个规则封装，才能让LLM**稳定调用、不报错、不破坏文件**：

## 1. 原子化：一个工具只做一件事
❌ 禁止：一个工具实现「读取+写入+筛选」
✅ 推荐：拆分独立工具（`excel_read`、`excel_write`、`excel_filter`）

## 2. 标准化：统一入参+出参
- **入参**：必传 `file_path`、`sheet_name`，可选 `range/value/condition`
- **出参**：固定返回JSON，包含：
  ```json
  {
    "success": true,
    "data": "操作结果",
    "message": "提示信息",
    "file_hash": "文件指纹（防篡改）"
  }
  ```

## 3. 安全化：无副作用 + 可回滚
- 所有写操作默认支持 `dry_run=True`（预览修改，不实际写入）
- 写操作前自动备份原文件
- 禁止直接修改原文件（操作副本，确认后替换）

## 4. 语义化：LLM能看懂
不用技术术语，用自然语言命名：
- 不用 `worksheet_append`，用 `excel_append_row`
- 不用 `get_used_range`，用 `excel_read_used_range`

## 5. 无代码：禁止暴露底层库
❌ 禁止：让Agent写 openpyxl 代码
✅ 推荐：封装好函数，Agent只传参数

# 三、极简封装示例（Python + Openpyxl）
直接把「读取Excel区域」封装成**Agent可调用的工具**，你可以照搬扩展：

```python
from openpyxl import load_workbook
import json

# 封装：读取Excel指定区域数据（原子工具）
def excel_read_range(file_path: str, sheet_name: str, cell_range: str) -> str:
    """
    AI Agent工具：读取Excel工作表的指定区域
    :param file_path: Excel文件路径
    :param sheet_name: 工作表名称
    :param cell_range: 单元格区域（如 A1:C3）
    :return: JSON格式结果（Agent可直接解析）
    """
    try:
        # 底层原理：加载文件 → 解析XML
        wb = load_workbook(file_path, read_only=True)
        ws = wb[sheet_name]
        
        # 读取数据
        data = []
        for row in ws[cell_range]:
            row_data = [cell.value for cell in row]
            data.append(row_data)
        
        wb.close()
        # 标准化返回
        return json.dumps({
            "success": True,
            "data": data,
            "range": cell_range,
            "message": "读取成功"
        }, ensure_ascii=False)
    
    except Exception as e:
        return json.dumps({
            "success": False,
            "error": str(e),
            "message": "读取失败"
        }, ensure_ascii=False)

# ==============================================
# Agent调用方式（LLM只需要传参数，不用写底层逻辑）
# ==============================================
result = excel_read_range(
    file_path="test.xlsx",
    sheet_name="Sheet1",
    cell_range="A1:C5"
)
print(result)
```

### 总结
1. **操作本质**：所有Excel操作都是**修改ZIP压缩包内的XML文件**
2. **核心操作**：围绕「文件→工作表→单元格→数据→公式」5个层级
3. **封装关键**：原子化、标准化、安全化、语义化，**不让LLM写底层代码**
4. **落地方式**：用openpyxl封装无头工具，用xlwings封装原生Excel工具，直接接入LangChain/AutoGen
