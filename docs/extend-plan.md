# 一、修改操作：原子化操作序列 + 一次性执行 + 单次 Diff

## 1. 现状痛点（你提到的核心问题）

- 零散调用 N 个修改工具 → 产生 N 份 Diff、N 次文件备份
- 多次修改叠加后，最终 Diff 无法对应原始指令意图
- 中间步骤失败会导致文件半修改（非原子）
- AI Agent 多轮调用成本高、易出错、日志混乱

## 2. 设计方案：新增「批量操作集」原子工具

### 核心理念

**对外只暴露一个批量修改入口，所有增删改操作打包成一个「操作序列」，原子执行：全成功 / 全失败，最终只返回 1 份总 Diff**。

完全符合你的场景：

> 大多数场景是根据指令**一次性修改**，而非交互式反复编辑

### 3. 架构落地（最小侵入，无重构）

1. **在 `types.rs` 定义统一的操作序列结构**
   - 支持所有现有修改操作：写单元格、写区域、插入行、删行、设置公式、合并单元格等
   - 纯数据结构，无业务逻辑
2. **在 `excel_write.rs` 新增一个原子函数：`execute_batch_operations`**
   - 内部**按顺序遍历执行**所有操作
   - 复用现有所有原子修改逻辑（不重复造轮子）
   - 基于 `modify_file` 实现**全内存修改**，最后一次性写入文件
   - 安全机制：**一次备份、一次指纹计算、一次回滚点**
3. **Diff 机制**
   - 所有操作执行完毕后，**只做一次全量 Diff**
   - 返回「总变更集」，清晰对应原始指令意图
4. **原子性保证**
   - 任何一步报错 → 立即终止 → 不写入文件 → 回滚到原始版本
   - 无中间状态，完美适配 Agent 任务执行

### 4. 对外接口（Agent 友好）

- CLI：`excel batch-modify --operations ops.json`
- HTTP：`POST /api/batch/modify`
- 入参：`file_path + sheet_name + dry_run + 操作序列数组`
- 出参：**唯一一份总 Diff** + 操作成功数 + 备份信息

### 5. 保留原有原子工具（不删除、不废弃）

- 简单场景（只改一个单元格）：继续用原子工具
- 复杂场景（多步骤修改）：强制用批量操作序列
- 架构兼容，灵活选择

### 6. 核心优势

✅ **Diff 极度清晰**：一次修改 → 一份 Diff，完全对齐指令意图
✅ **真正原子性**：全成功/全失败，无脏文件
✅ **性能提升**：一次文件 IO，一次备份，N 倍提速
✅ **Agent 更稳定**：一轮调用完成所有修改，无多轮幻觉
✅ **无架构入侵**：完全复用现有代码

# 二、批量数据修改：统一使用「指定范围 + 外部 CSV 文件」

## 1. 现状痛点

- 批量写入时，Agent 需要在工具参数中传递大量二维数组数值
- 参数体积巨大、序列化易失败
- LLM 极易写错数值、漏传、格式错乱
- 大表格（100行+）完全无法通过参数传值

## 2. 设计方案：**批量数据 = 范围定位 + CSV 文件数据源**

### 核心理念

**所有批量数据修改不再传递数值，只传递「目标单元格范围」+「CSV 文件路径」**
工具自动读取 CSV 并按位置写入 Excel。

### 3. 架构落地（极简）

1. **在 `excel_write.rs` 新增函数：`write_range_from_csv`**
   - 入参：`file_path, sheet_name, target_range, csv_path`
   - 逻辑：读取 CSV → 按行列对齐 → 写入指定 Excel 区域
   - 完全复用现有写入逻辑
2. **纳入批量操作序列**
   - CSV 写入作为一种标准操作，可放入 `batch_operations`
3. **CSV 规则（极简，Agent 易生成）**
   - 纯文本、逗号分隔
   - 行列与 Excel 目标区域严格一一对应
   - 无样式、无公式，纯数据

### 4. 对外接口

- CLI：`excel write-csv --file a.xlsx --sheet Sheet1 --range A1 --csv data.csv`
- HTTP：`POST /api/range/write-from-csv`
- 入参**无任何数值**，只有路径和位置

### 5. 强制适用边界

- **>5 个单元格** 的批量写入 → 必须用 CSV
- **单行/单个单元格** → 可继续用原生参数写入

### 6. 核心优势

✅ **Agent 调用极简**：LLM 只需要生成 CSV 路径 + 位置，不用写任何数值
✅ **无参数臃肿**：彻底解决大体积数据传输问题
✅ **极低错误率**：LLM 最擅长生成文件路径，不擅长生成长数组
✅ **支持超大批量**：万行数据也能稳定写入
✅ **格式标准化**：CSV 通用、易生成、易解析

# 三、两个方案整合后的终极工作流（AI Agent 最优）

这是你工具未来的**标准执行模式**，完全解决你提出的所有问题：

1. Agent 接收指令：`把Sheet1 A1:C10替换为数据，插入2行，设置D列公式`
2. Agent **生成 CSV 批量数据文件**
3. Agent **构造 1 个批量操作序列**（包含：CSV写入、插入行、设置公式）
4. 调用 **唯一批量修改接口**
5. 工具执行：
   - 一次备份
   - 内存中顺序执行所有操作
   - 原子性：全成功/全失败
   - 生成 **一份总 Diff**
6. 返回结果：
   - 成功/失败
   - 总变更 Diff（清晰、无冗余）
   - 备份路径
