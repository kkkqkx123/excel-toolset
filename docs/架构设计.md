# 二、整体设计原则
1. **扁平架构优先**：维持「入口层 + 核心能力层」两层极简架构，不新增中间转发/编排层，链路最短、维护成本最低。
2. **能力模块化内聚**：按「读、写、数据加工、VBA、Diff、通用安全」拆分独立模块，单一模块职责单一，互不耦合。
3. **通用组件全局复用**：文件操作、文件指纹、备份、快照、dry-run、Diff算法抽为公共组件，被所有业务能力复用。
4. **Diff能力双形态设计**：同时支持**独立对比命令/接口**、**写操作附属变更对比**，适配Agent不同使用场景。
5. **参数/结构统一**：全量CLI、HTTP、底层函数共用同一套DTO结构体，参数定义唯一，避免多套标准。
6. **兼容底层库特性**：适配`rust_xlsxwriter`「仅新建、不直接修改旧文件」的限制，统一使用「读原文件→新建文件→覆盖保存」的修改方案，逻辑内聚在写入模块。

延续轻量化两层结构，在上一版基础上**扩充模块、强化通用安全、新增Diff模块**，无额外抽象层。

┌───────────────────────────────▼─────────────────────────────────┐
│ 核心能力层（对内实现，库薄封装）                                 │
│ 由 通用基础模块 + 专项Excel能力模块 组成，纯函数集合，无状态     │

└─────────────────────────────────────────────────────────────────┘

## 分层详细职责
### 1. 入口层
分为两个并行子模块，**代码完全复用核心能力层**，仅做入口适配：
1. **CLI 子模块**
   - 基于 `clap` 实现**原子子命令**，每个Excel操作对应一条独立子命令
   - 解析命令行参数 → 调用核心层函数 → 格式化结果（纯文本/JSON）输出到终端
2. **HTTP 子模块**
   - 基于 `axum+tokio` 实现REST接口，**一个原子操作对应一个独立路由**
   - 解析JSON请求体 → 基础参数校验 → 调用核心层函数 → 按统一JSON格式返回响应
3. 公共能力（入口层通用）
   - 基础合法性校验（文件路径存在性、单元格格式、范围合法性）
   - 全局异常捕获、错误信息格式化

### 2. 核心能力层（纯函数集合，无类/服务/领域实体）
所有模块均为**薄封装**，直接对接`calamine`/`rust_xlsxwriter`/原生Rust标准库，按能力域拆分7个内聚模块，新增`security`安全模块、`excel_diff`差异模块，扩充原有读写/数据模块：
| 模块名          | 核心职责                                                                 | 底层依赖                |
|-----------------|--------------------------------------------------------------------------|-------------------------|
| `file_util`     | 基础文件IO、路径处理、临时目录管理、文件复制                             | Rust std                |
| `security`      | 全局安全组件：文件哈希(指纹)、自动备份、文件快照、回滚、dry-run状态管理    | file_util + std         |
| `excel_read`    | 所有**读取类**原子操作（文件信息、工作表、单元格、公式、样式、原数据读取） | calamine（只读库）|
| `excel_write`   | 所有**写入/修改类**原子操作（创建、单元格写入、行操作、公式/样式写入）| rust_xlsxwriter（只写库） |
| `excel_data`    | 纯内存数据加工：筛选、排序、去重、SQL查询（复用读写能力）| 依赖 excel_read         |
| `vba_util`      | VBA二进制流读写、导入/导出（仅透传，不解析/不执行）| calamine + rust_xlsxwriter |
| `excel_diff`    | Excel差异对比：文件级、工作表级、单元格区域级对比；生成结构化差异结果     | excel_read + 基础算法   |

> 关键说明：
> 1. 所有**写操作**会强制前置调用`security`模块（计算指纹、创建备份、标记dry-run）；
> 2. 所有**Diff能力**依赖`excel_read`读取两份文件数据，再做内存比对，不重复解析文件；
> 3. 数据处理类操作（筛选/排序等）基于`excel_read`读出数据后内存计算，再按需调用`excel_write`落地修改。

# 六、核心通用组件详细设计（安全 + Diff）
这是本次架构的**重点增强部分**，两个全局组件被所有原子能力复用，统一规则、统一实现。

## 1. 安全组件（security.rs）设计
目标：实现`文件指纹、自动备份、文件快照、dry-run、回滚`五大安全能力，**所有写操作强制前置调用**。
### 核心能力
1. **文件指纹（File Hash）**
   - 算法：SHA-256，对完整Excel文件计算哈希值；
   - 用途：防文件篡改、作为Diff的基准标识、返回体必带字段；
   - 时机：读文件/执行写操作前，自动计算并记录指纹。
2. **自动备份 & 快照**
   - 规则：任何写操作执行前，自动将原文件复制到临时备份目录（可配置路径），生成**版本快照**；
   - 快照元数据：关联原文件路径、操作时间、指纹、操作类型，支持按快照回滚。
3. **Dry-run（预执行）**
   - 入参：全局布尔参数 `dry_run: bool`（所有写操作统一携带）；
   - 逻辑：为`true`时，仅模拟执行、计算预期变更、生成Diff结果，**不落地文件修改**；
   - 输出：返回「模拟执行结果 + 预期Diff」，供Agent预览。
4. **文件回滚**
   - 能力：基于备份快照，一键恢复到操作前版本；
   - 独立命令/接口：`file rollback`、`/api/file/rollback`。

### 调用时机（固定链路）
所有写操作统一前置流程：`计算原文件指纹 → 创建备份快照 → 解析dry-run标记 → 执行业务逻辑`。

## 2. Diff差异组件（excel_diff.rs）设计
目标：支持**多粒度差异比对**，输出结构化数据，适配AI Agent解析；复用`excel_read`读取文件数据，不重复IO。
### 1）比对粒度（从粗到细，分级输出）
1. **文件级**：文件是否变更、工作表数量/名称差异；
2. **工作表级**：指定Sheet是否存在、行列总数差异；
3. **行级**：整行新增/删除/内容变更；
4. **单元格级**：单格内容、公式、样式变更（最小粒度）。

### 2）差异类型定义（结构化枚举）
统一标记变更类型，便于Agent识别：
- 新增（Add）、删除（Delete）、修改（Modify）、无变更（NoChange）

### 3）两种使用形态实现
1. **独立Diff查询**
   链路：接收两个文件路径 → `excel_read`分别读取两份文件全量数据 → 分级比对 → 组装结构化Diff结果 → 返回。
2. **写操作附属Diff**
   链路：写操作前读取原文件数据 → 执行修改逻辑（内存中生成新数据）→ 新旧数据比对 → 将Diff结果嵌入响应体`diff`字段。

### 4）性能优化
- 优先比对**文件指纹**：两份文件哈希一致则直接判定无差异，跳过全量解析；
- 大文件支持**区域限定比对**：仅解析指定Sheet/Range，减少内存占用。

# 七、全局入参/出参标准化规范
延续你原有规范，补充`dry_run`、`diff`、`file_hash`字段，**全CLI/HTTP/函数统一格式**。

## 1. 通用出参模板（所有接口/命令统一）
```json
{
  "success": true,          // 操作状态（布尔）
  "message": "操作描述",    // 文本提示
  "file_hash": "xxx",       // 当前文件SHA256指纹（必带）
  "data": {},               // 业务核心数据（读操作结果/写入结果）
  "diff": {},               // 【写操作专属】结构化差异数据（无变更则为null）
  "backup_info": {}         // 【写操作专属】备份快照信息（dry-run为true时无此字段）
}
```

## 2. 通用入参规则
1. **必传基础参数**：所有文件操作统一携带 `file_path`（文件路径）；涉及工作表必传 `sheet_name`；
2. **写操作通用可选参数**：统一携带 `dry_run: bool`（默认`false`）；
3. **Diff独立接口入参**：必传 `old_file_path`、`new_file_path`，可选限定`sheet_name`/`range`。

## 3. CLI输出规则
- 默认输出**纯JSON格式**（适配Agent程序解析）；
- 可选增加`--pretty`参数，格式化JSON便于人工查看。

# 八、核心业务流程链路（全场景）
基于分层架构，梳理5类高频核心流程，明确模块调用顺序。

### 流程1：纯读取类操作（例：读取单元格区域）
```
CLI/HTTP请求 → 入口层参数解析+基础校验 → excel_read（calamine读取数据）
→ 计算file_hash → 组装标准响应（data+file_hash）→ 返回结果
```
特点：无修改、无备份、无diff，链路最简单。

### 流程2：普通写操作（例：写入单元格，dry_run=false）
```
请求 → 入口层解析参数 → security（前置：计算原指纹 + 创建备份快照）
→ excel_write（读原文件 → 新建文件 → 写入数据）
→ excel_diff（新旧数据比对，生成变更diff）
→ 落地保存新文件 → 组装响应（data+hash+diff+backup_info）→ 返回
```

### 流程3：写操作（dry_run=true 预执行）
```
请求 → 入口层解析参数 → security（计算原指纹，不创建真实备份）
→ excel_write（内存模拟写入，不落地文件）
→ excel_diff（生成预期diff）
→ 组装响应（data+hash+diff，无backup_info）→ 返回
```
特点：仅预览，不修改原文件，安全风险为0。

### 流程4：独立Diff对比（两份文件比对）
```
请求 → 入口层解析双文件路径 → security（分别计算两个文件指纹，快速判同）
→ excel_read（读取两份文件数据）→ excel_diff（分级比对）
→ 组装响应（diff为核心数据）→ 返回
```

### 流程5：VBA读写流程
```
导出VBA：请求 → excel_read/vba_util（calamine读取VBA二进制）→ 组装结果返回
导入VBA：请求 → security（备份+指纹）→ vba_util/excel_write（嵌入二进制流）
→ 计算diff → 保存文件 → 返回结果
```
特点：仅透传二进制，不解析、不执行VBA。

# 九、扩展与迭代设计
基于当前架构，后续新增功能可按统一规则扩展，**不改动现有分层与目录**：
1. **新增原子操作**
   - 第一步：在`types.rs`补充对应请求/响应DTO；
   - 第二步：在对应能力模块（read/write/data）新增独立函数；
   - 第三步：在CLI/HTTP入口新增子命令/路由；
   - 全程不新增目录、不改动架构。
2. **增强Diff能力**
   - 仅迭代`excel_diff.rs`模块，优化比对算法、新增差异粒度，对外接口格式不变。
3. **新增文件格式支持**
   - 依托`calamine`原生能力，仅扩展函数参数与解析逻辑，上层入口无感知。
4. **部署扩展**
   - HTTP服务支持多实例、容器化部署；CLI单二进制编译，跨平台无依赖，架构天然适配。

# 十、架构最终优势总结
1. **轻量化适配定位**：完全摒弃重型后端架构，回归「库薄封装工具」本质，代码精简、上手快、维护成本低。
2. **能力全覆盖**：完整承载你定义的所有Excel原子操作、VBA、Diff能力，原子拆分符合AI Agent工具设计要求。
3. **安全体系完善**：指纹、备份、快照、dry-run、回滚全链路覆盖，避免文件损坏/数据丢失。
4. **Diff双形态灵活**：既支持独立文件对比，也支持写操作自动返回变更，适配Agent多样化使用场景。
5. **职责边界清晰**：本项目专注「函数实现+参数+入口」，上层AI Agent专注「工具Schema+LLM编排」，分层解耦。
6. **扩展性强**：新增功能遵循统一规则，无需重构架构，长期迭代稳定。
7. **底层依赖稳定**：基于`calamine+rust_xlsxwriter`无头方案，跨平台、无Office依赖，适配服务端/容器部署。
