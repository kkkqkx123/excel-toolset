# 一、核心技术选型：直接给结论（拒绝复杂）
## 最终方案
**主方案：追踪工具调用历史 + 持久化（优先）**
**辅方案：Excel解析库+diff工具（兜底校验）**
✅ 这是**最简单、前端渲染成本最低、最适配LLM编辑Excel**的方案，完胜纯「解析库+diff」。

---

## 两种方案详细对比（告诉你为什么这么选）
### 方案1：纯 Excel解析库 + diff工具（如xlsx.js + deep-diff）
**原理**：保存「修改前Excel」+「修改后Excel」→ 前端解析两个文件 → 自动对比生成diff
**适用场景**：用户手动修改Excel、无操作日志、纯本地diff
**缺点（致命，不适合LLM场景）**
1. **前端渲染极麻烦**：需要解析二进制Excel，处理多sheet、格式、公式，代码量大；
2. **diff冗余**：会把「公式自动计算的值变化」当成修改，diff列表爆炸；
3. **无意图**：只知道「改了什么」，不知道「LLM为什么改」；
4. **性能差**：大文件解析卡顿。

### 方案2：追踪工具调用历史 + 持久化（LLM专属最优解）
**原理**：LLM编辑Excel的本质是**执行代码/API指令**（如`修改A2=200`、`新增公式C5=SUM(C2:C4)`），直接**记录这些指令并存储**，前端直接渲染指令=diff
**优点（完美匹配你的需求）**
1. **前端极简**：不用解析Excel，直接渲染JSON格式的操作历史，几行代码搞定表格；
2. **diff精准**：只记录LLM**主动修改**，自动过滤公式级联计算的被动变化；
3. **可观测性拉满**：知道LLM每一步操作意图，不是黑盒；
4. **轻量化**：持久化存JSON（几KB），远比重存Excel文件高效。

### 方案3：混合架构（最终落地版）
- **写入时**：强制记录LLM的**工具调用历史**（主动操作），持久化存数据库/本地；
- **校验时**：用轻量解析库（如xlsx.js）解析最终Excel，**仅校验数据值**，防止LLM执行出错；
- **前端渲染**：直接用「工具调用历史」渲染diff表格，零难度。

# 二、公式的标准化处理方案（简单、前端友好）
## 核心原则
**不单独用文本描述，也不只用值**，采用「**双字段展示 + 分离式diff**」，完全适配前端表格：

### 1. 单元格存储结构（持久化+前端渲染用）
每个公式单元格，固定存2个值（极简结构）：
```json
{
  "cell": "Sheet1!C5",
  "type": "formula", // 区分普通值/公式
  "formula": "=SUM(C2:C4)", // 公式本体
  "value": 300, // 公式计算结果
  "diffType": "modify" // 新增/修改/删除
}
```

### 2. 前端表格渲染规则（简单直观）
| 单元格位置 | 类型 | 旧值/旧公式 | 新值/新公式 | 状态标记 |
|------------|------|-------------|-------------|----------|
| Sheet1!C5 | 公式 | 公式：=AVERAGE(C2:C4) <br> 值：150 | 公式：=SUM(C2:C4) <br> 值：300 | ✏️ 修改 |
| Sheet1!A2 | 数值 | 100 | 200 | ✏️ 修改 |

- **默认展示：计算值**（和Excel一致）；
- **hover/切换按钮：展示公式**（前端一行CSS实现）；
- **diff报告：公式和值同时标注**（用户一眼看懂：改了公式 → 值变了）。

### 3. 文本描述规范（人类可读，极简）
统一格式，前端直接拼接：
`【Sheet1!C5】公式修改：=AVERAGE(C2:C4) → =SUM(C2:C4)，计算值自动更新：150 → 300`

# 三、多公式关联（级联计算）的处理方案（解决核心痛点）
## 问题场景
`A5=SUM(A2:A4)` → `B5=A5*10` → `C5=B5+100`
你修改**A2**，LLM只动了A2，但A5/B5/C5的值都会自动变；
如果不处理，diff会把4个单元格都标记为修改，**冗余且混乱**。

## 极简解决方案（前端零复杂度）
### 核心规则：只追踪「主动修改」，标记「被动更新」
1. **源头区分**
   - ✅ **主动修改**：LLM直接编辑的单元格（如A2）→ **高亮红色**，计入diff核心列表；
   - ⚠️ **被动更新**：公式关联自动计算的单元格（A5/B5/C5）→ **高亮黄色**，折叠展示/备注；
2. **不存储被动修改**
   持久化只存LLM的**主动操作**，被动值变化由Excel公式自动计算，**不写入diff历史**；
3. **前端轻量提示**
   在主动修改的单元格下方，加一行小字：
   `关联影响：A5、B5、C5值自动更新`

### 进阶（可选，简单实现）
如果需要展示依赖关系，前端用**极简依赖链**：
`A2 → A5(SUM) → B5(计算) → C5(计算)`
不用构建复杂依赖图，**仅文本展示**即可满足可观测性。

# 五、最终总结（3句话记住）
1. **选型**：优先用 **LLM工具调用历史+持久化**，前端渲染最简单；解析库仅兜底校验，不做主diff方案；
2. **公式**：前端**值+公式双展示**，diff同时记录公式修改和值变化，文本描述标准化；
3. **多公式关联**：**只标记LLM主动修改**，被动计算的变化仅做提示，不冗余写入diff，保持简洁。

这个方案完全满足你**简单前端渲染 + 类CSV diff可观测性**的需求，轻量化、易开发、无坑！
